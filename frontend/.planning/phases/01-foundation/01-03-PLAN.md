**Fix 1 — ProtectedRoute.tsx:** Replace `withAuthenticationRequired` HOC with a custom guard component that uses `useAuth0()` directly. The new logic:
- If `isLoading` is true, render `<LoadingScreen />`
- If `error` exists, render `<AuthErrorFallback error={error} />`
- If `isAuthenticated` is false, render `<Navigate to="/login" replace />` (from react-router)
- If `isAuthenticated` is true, render `<Outlet />` (or the child component)

Why NOT `withAuthenticationRequired`: That HOC calls `loginWithRedirect()` automatically for unauthenticated users, bypassing the /login page entirely. The user's locked decision is "Custom branded landing page with app name and Login button for unauthenticated users (not auto-redirect)".

Rewrite ProtectedRoute as a wrapper route component (not HOC-based):
```typescript
import { useAuth0 } from '@auth0/auth0-react';
import { Navigate, Outlet } from 'react-router';
import { LoadingScreen } from '../components/ui/LoadingScreen';
import { AuthErrorFallback } from '../components/ui/AuthErrorFallback';

export function ProtectedRoute() {
  const { isAuthenticated, isLoading, error } = useAuth0();

  if (isLoading) return <LoadingScreen />;
  if (error) return <AuthErrorFallback error={error} />;
  if (!isAuthenticated) return <Navigate to="/login" replace />;

  return <Outlet />;
}
```

**Fix 2 — App.tsx:** Update routing to use ProtectedRoute as a layout route wrapper (no more `component` prop):
```typescript
<Routes>
  <Route path="/login" element={<LoginPage />} />
  <Route element={<ProtectedRoute />}>
    <Route path="/" element={<AppLayout />}>
      <Route index element={<HomePage />} />
    </Route>
  </Route>
</Routes>
```

**Fix 3 — AuthErrorFallback.tsx:** Create a new component that displays Auth0 errors in a user-friendly way:
- Centered on screen (same layout style as LoginPage: flex center, minHeight 100vh, dark background)
- MUI Paper card showing:
  - Typography h5: "Erro de Autenticacao"
  - Typography body1 color text.secondary: The error message from Auth0 (error.message)
  - Button "Tentar Novamente" that calls `loginWithRedirect()` from useAuth0
  - Button "Voltar" (secondary/text variant) that navigates to /login
- Props: `{ error: Error }`

This ensures that if Auth0 returns "unauthorized_client: Grant type 'authorization_code' not allowed" (or any other error), the user sees a readable message instead of a blank screen.
