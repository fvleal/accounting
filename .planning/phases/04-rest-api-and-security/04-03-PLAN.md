  <action>
    Create `src/account/interface/controllers/account.controller.ts` with @Controller('accounts').

    Constructor injects all 9 use cases: CreateAccountCommand, UpdateNameCommand, UpdatePhoneCommand, UpdateBirthDateCommand, UploadAccountPhotoCommand, GetAccountByIdQuery, GetMeQuery, FindAccountByFieldQuery, ListAccountsQuery.

    IMPORTANT: Declare routes in this exact order to avoid the /me vs /:id route conflict (Pitfall 1):

    1. **POST /** -- @Post(), @HttpCode(201), @Roles('create:account')
       - Params: @CurrentUser() user: JwtPayload, @Body() dto: CreateAccountDto
       - Calls createAccount.execute({ auth0Sub: user.sub, email: user.email, name: dto.name, cpf: dto.cpf })
       - Returns AccountResponseDto.fromOutput(output)

    2. **GET /me** -- @Get('me'), @Roles('read:own-account')
       - Params: @CurrentUser() user: JwtPayload
       - Calls getMe.execute({ auth0Sub: user.sub })
       - Returns AccountResponseDto.fromOutput(output)

    3. **GET /:id** -- @Get(':id')
       - Params: @Param('id', ParseUUIDPipe) id: string
       - Calls getById.execute({ id })
       - Returns AccountResponseDto.fromOutput(output)
       - No @Roles -- any authenticated user can fetch by ID (per CONTEXT.md: user role for getMe, admin for getByCPF; getById is not explicitly restricted)

    4. **GET /** -- @Get(), @Roles('read:accounts')
       - Params: @Query() query: ListAccountsQueryDto
       - If query.cpf is present: call findByField.execute({ field: 'cpf', value: query.cpf }), return AccountResponseDto.fromOutput(output)
       - If query.cpf is absent: call listAccounts.execute({ limit: query.limit ?? 20, offset: query.offset ?? 0 }), return { data: output.data.map(AccountResponseDto.fromOutput), total: output.total, offset: query.offset ?? 0, limit: query.limit ?? 20 }
       - The interceptor will detect the list shape and build appropriate meta

    5. **PATCH /:id** -- @Patch(':id')
       - Params: @Param('id', ParseUUIDPipe) id: string, @Body() dto: UpdateAccountDto
       - If dto.name is present: call updateName.execute({ accountId: id, name: dto.name })
       - If dto.birthDate is present: call updateBirthDate.execute({ accountId: id, birthDate: dto.birthDate })
       - Return AccountResponseDto.fromOutput(lastOutput) -- use the output from the last command executed
       - If neither field present: throw BadRequestException('At least one field must be provided')

    6. **POST /:id/phone/send-code** -- @Post(':id/phone/send-code')
       - Params: @Param('id', ParseUUIDPipe) id: string, @Body() dto: SendPhoneCodeDto
       - Calls updatePhone.execute({ accountId: id, phone: dto.phone })
       - Returns AccountResponseDto.fromOutput(output)
       - NOTE: This wires to UpdatePhoneCommand directly (phone verification not yet implemented). Sets phone with phoneVerified=false.

    7. **POST /:id/phone/verify** -- @Post(':id/phone/verify')
       - Params: @Param('id', ParseUUIDPipe) id: string, @Body() dto: VerifyPhoneDto
       - Throws HttpException with status 501 and message 'Phone verification not yet implemented'
       - NOTE: VerifyPhoneCommand does not exist. This endpoint is a placeholder per RESEARCH.md recommendation.

    8. **POST /:id/photo** -- @Post(':id/photo'), @UseInterceptors(FileInterceptor('file'))
       - Params: @Param('id', ParseUUIDPipe) id: string, @UploadedFile(new ParseFilePipe({ validators: [new MaxFileSizeValidator({ maxSize: 5 * 1024 * 1024 })] })) file: Express.Multer.File
       - Calls uploadPhoto.execute({ accountId: id, buffer: file.buffer, contentType: file.mimetype })
       - Returns AccountResponseDto.fromOutput(output)

    Import types: use `import type { JwtPayload }` for the interface. Import decorators/guards from auth barrel export. Import DTOs from ../dtos/. Import use cases from ../../application/.
