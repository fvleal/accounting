    - Email.create("invalid") throws "Invalid email format"
    - Email.create("") throws
    - Email.create("@missing.com") throws
    - Two Emails with same address are equal
    - value getter returns the normalized email string

    CPF (using cpf-cnpj-validator):
    - CPF.create("123.456.789-09") succeeds (valid masked CPF -- use a real valid CPF for testing)
    - CPF.create("12345678909") succeeds (valid unmasked)
    - CPF.create("111.111.111-11") throws (known invalid -- all same digits)
    - CPF.create("000.000.000-00") throws
    - CPF.create("12345") throws (too short)
    - CPF.create("") throws
    - value getter returns digits only (no mask)
    - formatted getter returns masked format
    - Two CPFs with same digits are equal

    PhoneNumber:
    - PhoneNumber.create("11999887766") succeeds (mobile with 9)
    - PhoneNumber.create("1133445566") succeeds (landline 8 digits)
    - PhoneNumber.create("(11) 99988-7766") succeeds (strips mask)
    - PhoneNumber.create("123") throws (too short)
    - PhoneNumber.create("0099887766") throws (DDD cannot start with 0)
    - PhoneNumber.create("") throws
    - value getter returns digits only

    PersonName:
    - PersonName.create("John Doe") succeeds
    - PersonName.create("Maria da Silva") succeeds (3+ words)
    - PersonName.create("John") throws (single word)
    - PersonName.create("") throws
    - PersonName.create("  ") throws
    - value getter returns the full name

    BirthDate:
    - BirthDate.create(new Date("1990-01-15")) succeeds (past date)
    - BirthDate.create(new Date("2000-12-31")) succeeds
    - BirthDate.create(futureDate) throws (date in the future)
    - value getter returns the Date
    - Two BirthDates with same date are equal
    - NOTE: Store date as ISO string internally (avoid Date mutability in frozen props)
  </behavior>
  <action>
RED: Write spec files for all 5 VOs first. Each VO spec should test valid construction, invalid input rejection, getter behavior, and equality. Use real valid CPF numbers for testing (e.g., "529.982.247-25" is commonly used in Brazilian test suites). Run tests -- all must fail.

GREEN: Implement each VO following RESEARCH.md patterns and user decisions:

- Email: extends ValueObject<{value: string}>. Static create() normalizes to lowercase and trims. validate() checks regex /^[^\s@]+@[^\s@]+\.[^\s@]+$/. Private constructor calls super(props).

- CPF: extends ValueObject<{value: string}>. Static create() strips non-digits before constructing. validate() uses `cpf.isValid()` from cpf-cnpj-validator. Getter `formatted` uses `cpf.format()`. Import as: `import { cpf as cpfValidator } from 'cpf-cnpj-validator';`

- PhoneNumber: extends ValueObject<{value: string}>. Static create() strips non-digits. validate() checks regex /^[1-9]{2}[2-9]\d{7,8}$/ (DDD 2 digits starting 1-9, number starts 2-9, total 10-11 digits).

- PersonName: extends ValueObject<{value: string}>. Static create() trims whitespace. validate() checks that trimmed value split by whitespace has >= 2 words.

- BirthDate: extends ValueObject<{value: string}> (store as ISO string to avoid Date mutability). Static create() accepts Date, converts to ISO string. validate() checks the date is not in the future. Getter returns new Date(this.props.value).

Create `src/account/domain/value-objects/index.ts` barrel file exporting all 5 VOs.

REFACTOR: Ensure consistent error messages, clean imports.

IMPORTANT: All VOs import from `src/shared/domain/value-object.base.ts` only. NO @nestjs imports. CPF is the ONLY VO with an external dependency (cpf-cnpj-validator), which is allowed in domain per user decision.
