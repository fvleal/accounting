  <name>Task 2: AccountMapper and PrismaAccountRepository with transactional events</name>
  <files>src/account/infrastructure/mappers/account.mapper.ts, src/account/infrastructure/adapters/prisma-account.repository.ts</files>
  <action>
1. Create `src/account/infrastructure/mappers/account.mapper.ts`:
   - Static class `AccountMapper` (not injectable) with two static methods:
   - `toDomain(raw: PrismaAccountType): Account` -- calls `Account.reconstitute(raw.id, { name: raw.name, email: raw.email, cpf: raw.cpf, birthDate: raw.birthDate, phone: raw.phone, photoUrl: raw.photoUrl, createdAt: raw.createdAt, updatedAt: raw.updatedAt })`. Import the Prisma generated Account type as `PrismaAccountType` (or use `import { Account as PrismaAccount } from '../../../generated/prisma/client.js'` and alias to avoid conflict with domain Account).
   - `toPersistence(account: Account)` -- returns plain object with snake_case DB column names: `{ id: account.id, name: account.name, email: account.email, cpf: account.cpf, birth_date: account.birthDate, phone: account.phone, phone_verified: false, photo_url: account.photoUrl, created_at: account.createdAt, updated_at: account.updatedAt }`.
   - NOTE on snake_case: Prisma uses the @map field names internally for the DB, but the TypeScript types use camelCase (birthDate, phoneVerified, etc.). So toPersistence should return camelCase keys matching the Prisma TypeScript model, NOT snake_case DB column names. Prisma handles the @map translation. Return: `{ id, name, email, cpf, birthDate, phone, phoneVerified: false, photoUrl, createdAt, updatedAt }`.

2. Create `src/account/infrastructure/adapters/prisma-account.repository.ts`:
   - @Injectable() class PrismaAccountRepository implements AccountRepositoryPort
   - Constructor injects PrismaService and EventEmitter2
   - `save(account)`: Get persistence data via AccountMapper.toPersistence(), get events via account.getEvents(). Use this.prisma.$transaction(async (tx) => { upsert with where: { id: data.id }, create: data, update: data; then for each event, emit via this.eventEmitter.emit(event.constructor.name, event) }). After transaction, call account.clearEvents().
   - `findById(id)`: prisma.account.findUnique({ where: { id } }), map with AccountMapper.toDomain or return null
   - `findByEmail(email)`: prisma.account.findUnique({ where: { email } }), map or null
   - `findByCpf(cpf)`: prisma.account.findUnique({ where: { cpf } }), map or null
   - `findAll(params)`: Use prisma.$transaction([ prisma.account.findMany({ skip: params.offset, take: params.limit, orderBy: { createdAt: 'desc' } }), prisma.account.count() ]). Return { data: rows.map(AccountMapper.toDomain), total }.
   - `exists(id)`: prisma.account.count({ where: { id } }), return count > 0

   IMPORTANT: Use `event.constructor.name` for EventEmitter2 event name since DomainEvent has no eventName property. This emits "AccountCreated" or "AccountUpdated" as event names.
  </action>
