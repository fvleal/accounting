  <action>
1. Run `npx @nestjs/cli@11 new account-service --strict --package-manager npm` in the project root. Since we are already in a directory, scaffold into a temp folder and move contents up, OR use `nest new . --strict` if supported. The goal is the project files live at the repo root (C:/Users/felip/Desktop/gsd-learning/).

2. Remove Jest and its config:
   ```
   npm uninstall jest @types/jest ts-jest
   ```
   Delete `jest` config from package.json if present. Delete any jest.config or jest-e2e config files.

3. Install Vitest + SWC stack:
   ```
   npm install --save-dev vitest unplugin-swc @swc/core vite-tsconfig-paths @vitest/coverage-v8
   ```

4. Create `vitest.config.ts` at project root:
   ```typescript
   import swc from 'unplugin-swc';
   import { defineConfig } from 'vitest/config';
   import tsconfigPaths from 'vite-tsconfig-paths';

   export default defineConfig({
     plugins: [
       tsconfigPaths(),
       swc.vite({
         module: { type: 'es6' },
       }),
     ],
     test: {
       globals: true,
       root: './',
     },
   });
   ```

5. Update package.json scripts:
   - `"test": "vitest run"`
   - `"test:watch": "vitest"`
   - `"test:cov": "vitest run --coverage"`

6. Install domain dependency:
   ```
   npm install cpf-cnpj-validator
   ```

7. Create a trivial test `src/app.spec.ts` that imports AppService and verifies getHello() returns the expected string, confirming Vitest + SWC + NestJS decorators work together.

8. Verify NestJS boots: `npx ts-node -e "console.log('boot check')"` or `npm run build` succeeds.

IMPORTANT: Keep the default NestJS app.module, app.controller, app.service intact -- they prove the framework boots. Do NOT delete @nestjs/testing -- it will be needed in Phase 5.
  </action>
  <verify>
    <automated>npm test -- --reporter=verbose 2>&1 | tail -20</automated>
  </verify>

   - Include standard typescript-eslint recommended rules

   Use the exact config from RESEARCH.md ESLint section as the starting point.

7. After confirming boundaries work, delete the `_boundary-test.ts` file.

8. Update package.json lint script if needed to use the new config format.
