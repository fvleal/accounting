  <behavior>
    AccountCreated event:
    - Extends DomainEvent
    - Carries full account snapshot: id, name, email, cpf, birthDate (if present), phone (if present), photoUrl (if present), createdAt
    - occurredOn is set automatically

    AccountUpdated event:
    - Extends DomainEvent
    - Carries accountId and a changes object with before/after for each changed field
    - occurredOn is set automatically

    Account.create():
    - Accepts { name: string, email: string, cpf: string } as required fields
    - Generates UUID via crypto.randomUUID()
    - Creates Value Objects internally (Email, CPF, PersonName) -- validates at construction
    - Collects exactly one AccountCreated event
    - Returns Account instance with generated ID
    - Throws if name is invalid (single word)
    - Throws if email is invalid
    - Throws if CPF is invalid

    Account.reconstitute():
    - Accepts id + all props (including optional birthDate, phone, photoUrl)
    - Creates Value Objects internally -- validates
    - Does NOT generate a new ID
    - Does NOT collect any events
    - Returns Account with the provided ID

    Account getters:
    - id, name (PersonName), email (Email), cpf (CPF) always present
    - birthDate (BirthDate | null), phone (PhoneNumber | null), photoUrl (string | null) nullable
    - createdAt (Date), updatedAt (Date)

    Account.updateName(newName: string):
    - Creates new PersonName VO (validates)
    - Collects AccountUpdated event with { name: { before: oldName, after: newName } }
    - Updates internal state

    Account.updatePhone(phone: string):
    - Creates new PhoneNumber VO (validates)
    - Collects AccountUpdated event with { phone: { before: oldPhone, after: newPhone } }

    Account.updateBirthDate(date: Date):
    - Creates new BirthDate VO (validates)
    - Collects AccountUpdated event with { birthDate: { before: old, after: new } }

    Account.updatePhoto(url: string):
    - Validates URL format (simple check)
    - Collects AccountUpdated event with { photoUrl: { before: old, after: new } }

    AccountRepositoryPort:
    - Interface (not class) with methods: save(account: Account): Promise<void>, findById(id: string): Promise<Account | null>, findByEmail(email: string): Promise<Account | null>, findByCpf(cpf: string): Promise<Account | null>, exists(id: string): Promise<boolean>
    - TypeScript compilation confirms it exists (no runtime test needed for interface)
  </behavior>

1. **AccountCreated event** (`account-created.event.ts`): extends DomainEvent. Constructor accepts a snapshot object with all account fields (id, name, email, cpf, birthDate?, phone?, photoUrl?, createdAt). Store as public readonly properties.

2. **AccountUpdated event** (`account-updated.event.ts`): extends DomainEvent. Constructor accepts accountId and a changes record: `Record<string, { before: unknown; after: unknown }>`. Store as public readonly properties.

3. **Account aggregate** (`account.entity.ts`): extends AggregateRoot<string>.
   - Private fields: _name (PersonName), _email (Email), _cpf (CPF), _birthDate (BirthDate | null), _phone (PhoneNumber | null), _photoUrl (string | null), _createdAt (Date), _updatedAt (Date)
   - Private constructor(id, props) sets all fields via private setters. Each private setter validates its field (creates VO or checks URL format) before assignment.
   - Static create({ name, email, cpf }): generates UUID via `randomUUID()` from 'node:crypto', constructs via private constructor, calls `this.addEvent(new AccountCreated(...))` with full snapshot, returns account.
   - Static reconstitute(id, props): constructs via private constructor, does NOT add events, returns account. Props include all fields (required + optional including createdAt, updatedAt).
   - Public getters for all fields. Name getter returns `this._name.value`, email returns `this._email.value`, cpf returns `this._cpf.value`, etc. OR return the VO instances (prefer returning primitive strings for simpler consumption -- return VO.value).
   - updateName(name: string): creates PersonName.create(name), stores old value, updates, adds AccountUpdated event with before/after.
   - updatePhone(phone: string): creates PhoneNumber.create(phone), stores old, updates, adds event.
   - updateBirthDate(date: Date): creates BirthDate.create(date), stores old, updates, adds event.
   - updatePhoto(url: string): validates URL with `new URL(url)` (throws if invalid), stores old, updates, adds event.
   - Each update method also updates _updatedAt to new Date().

4. **AccountRepositoryPort** (`account.repository.port.ts`): TypeScript interface, not a class. Methods: save, findById, findByEmail, findByCpf, exists. Import Account type.
